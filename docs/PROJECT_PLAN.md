# IPCSharedDataPool 项目计划

## 项目信息

| 项目 | 内容 |
|------|------|
| 项目名称 | IPCSharedDataPool - 电力系统进程间共享数据池与事件中心 |
| 创建日期 | 2026-03-04 |
| 技术栈 | C++17, POSIX (shm, pthread, semaphore) |
| 依赖 | pugixml（核心库XML解析），Qt5（仅测试程序） |

| 优先级 | 功能 | 状态 | 说明 |

| **🟠 P1** | IEC 61850 模型 | ✅ 已完成 | 现代变电站标准要求 |
| **🟡 P2** | GOOSE 支持 | 🔲 待开始 | 如果需要跳闸功能则升为P0 |
| **🟡 P2** | 健康监控 | 🔲 待开始 | 运维必备 |
| **🟢 P3** | 历史归档 | 🔲 待开始 | 高级功能 |

## 代码审查修复记录 (2026-03-05)

### 已修复的高优先级问题

| 问题 | 修复方式 | 影响 |
|------|----------|------|
| 🔴 ProcessInfo 包含 atomic 成员 | 改用 `volatile bool` + 代码同步 | 共享内存兼容性 |
| 🔴 updateHeartbeat 未加锁 | 使用 `__sync_lock_test_and_set` 原子操作 | 数据竞争 |
| 🟡 DZ/YK 操作缺少统计计数 | 添加 `incrementWriteCount()` | 统计不完整 |
| 🟡 批量操作静默失败 | 新增 `successCount` 输出参数 | 调试困难 |
| 🟡 校验和算法简单 | 改用 CRC32 算法 | 数据完整性 |
| 🟢 快照非原子写入 | 使用临时文件 + rename 模式 | 可靠性 |

### 已修改文件

- `Common.h`: ProcessInfo 改用 `volatile bool`，CRC32 校验和
- `SharedDataPool.h`: 批量操作接口增加 `successCount` 参数
- `SharedDataPool.cpp`: 原子心跳、统计计数、原子快照写入
- `tests/*.cpp`: 适配 ProcessInfo::active 改动
